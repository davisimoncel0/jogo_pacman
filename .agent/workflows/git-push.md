# Git Push

1. Utilize sempre a rule language.md
2. Verifique o que foi alterado, correcoes e etc
3. Adicione os arquivos e faca os commits analisando cada alteracao e usando o git sematico (chore, feat, refactor e etc)
4. Apos todas as mensagens de commits e arquivos prontos, faca o push
