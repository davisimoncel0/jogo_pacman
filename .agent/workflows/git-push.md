# Git Push

1. Utilize sempre a rule linguagem.md
2. Se nao tiver nada para commit e push, nao faca nada, aborte, sucesso.
3. Verifique o que foi alterado, correcoes e etc
4. Faca os commits individuais analisando cada arquivo e suas devidas alteracoes
5. Use o git sematico (chore, feat, refactor e etc) nos commits
6. Apos todas as mensagens de commits e arquivos prontos, faca o push
